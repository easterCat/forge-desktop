# Forge 设计规范

> 本文档为 Forge 产品的**高层级设计规范**，用于明确设计意图、风格基调与边界约束，**不包含具体实现细节**。
> 具体色值、尺寸、组件样式、文件结构等内容参见 `design/tokens/`（设计 Token 库）与 `design/components/`（组件实现规范）。
>
> 跨平台玻璃态视觉基线以 `design/forge-cross-platform-glass.html`（v3 原型）为参照实现。

---

## 1. 设计哲学

Forge 的视觉语言是**跨平台玻璃态 × 临床化数据**的融合：以多层次半透明玻璃面承载高密度操作面板，以瑞士字体排版的严谨性约束数据呈现，以**单一深色（或近黑）文字 + 主题驱动色板**建立清晰的信息层级，整体呈现**跨平台一致的现代操作台体验**。

核心立场：

- **玻璃是结构，不是装饰** — `backdrop-filter: blur()` 必须服务于信息层级（窗口 > 侧栏 > 卡片 > 输入 > 内嵌元素），任何一层玻璃必须有功能理由
- **信息层级 > 装饰表达** — 一切视觉决策以信息可读性为第一性原则
- **数据即界面** — 面向操作者（operator）的工具型产品，非消费类产品
- **跨平台一致** — 同一份设计在 macOS / Windows / Linux 桌面与移动端呈现一致的语义与节奏，**不因平台差异而妥协**
- **克制优先** — 默认状态即终态，交互反馈即动画的唯一触发源；数据卡允许的持续动画**仅限极慢速、低对比度的氛围层**（见 §7.5）

## 2. 主题系统

产品提供**多主题并行**机制，由用户按场景与偏好切换；产品**至少预置 20 个主题**（参见 `design/tokens/themes/`），覆盖冷暖、明暗、节日与行业场景。

### 2.1 主题分类

| 类别         | 数量 | 代表主题                                                       | 视觉基调           |
| ------------ | ---- | -------------------------------------------------------------- | ------------------ |
| **Warm 系**  | 5    | `warm`(默认)、`sakura`、`ember`、`desert`、`rose-gold`         | 暖色调、柔和玻璃态 |
| **Cool 系**  | 5    | `cool-mist`、`arctic`、`nordic`、`ocean`、`aurora`             | 冷色调、清爽玻璃态 |
| **Nature 系**| 4    | `sage`、`forest`、`peach-fuzz`、`cream`                        | 自然色、低饱和     |
| **Mood 系**  | 4    | `lavender`、`cotton-candy`、`slate`、`cyberpunk`               | 表达性、强情绪     |
| **Dark 系**  | 2    | `midnight`、`charcoal`                                         | 暗色玻璃态         |

### 2.2 主题边界

- 主题**不混合使用**于同一界面或同一组件实例
- 主题切换**全局生效**，不允许单页面或单组件的主题覆盖
- 每个主题**均需通过完整无障碍审计**（对比度、焦点指示、色盲友好）
- 每个主题**必须**为浅色 / 暗色两种基线之一提供可工作版本（暗色基线不强制同名主题）
- 主题相关的具体 Token（色值、阴影、圆角、玻璃透明度）由 `design/tokens/` 维护，本规范不固化数值

## 3. 色彩角色定义

色彩规范描述的是**角色（role）**而非**色值（value）**。具体色值由设计 Token 维护。

### 3.1 通用色彩角色

- **核心表面色** — 划分背景层级，从外壳（`--bg`）到三级元素
- **玻璃面层级色** — 5 级玻璃面（`--glass-window` / `--glass-sidebar` / `--glass-topbar` / `--glass-bg` / `--glass-input`），透明度随层级递减
- **文字层级色** — 主标题 → 正文 → 次级信息 → 占位符，至少 4 级（`--fg-title` / `--fg` / `--fg-muted` / `--fg-ghost`）
- **主强调色** — 每个主题**仅一种**，用于激活状态、关键操作、焦点环
- **状态语义色** — 成功 / 错误 / 警告 / 信息，各主题独立映射
- **Tint 氛围色** — 4 种低饱和度氛围色（`warm` / `cool` / `soft` / `amber`），用于数据卡内部氛围层与边框着色

### 3.2 边界约束

- 禁止使用纯黑 `#000000` 与纯白 `#FFFFFF` 作为表面色
- 禁止使用渐变背景填充容器（**例外**：主背景允许使用多色径向渐变 + 网格纹理组合，但仅限 `body` 元素；其他所有容器保持纯色或玻璃面）
- 禁止在单一界面中使用超过一个主强调色
- 状态语义色**必须**配合图标或文字标签使用，不得仅靠颜色传达含义
- 文字与背景对比度：正文 ≥ 4.5:1，关键 UI ≥ 7:1
- Tint 氛围色**不得**作为强调色使用，仅作氛围层；其不透明度不超过 0.20

## 4. 字体角色定义

### 4.1 字体族（v3 锁定）

- **西文/UI 字体族** — `Inter`（字重 400 / 500 / 600 / 700）
- **中文字体族** — 系统字体回退链（`PingFang SC` / `Microsoft YaHei` / `Noto Sans SC`），与 Inter 视觉权重协调
- **等宽字体族** — `JetBrains Mono`（字重 400 / 500 / 600 / 700），回退 `SF Mono` / `Fira Code` / `Menlo` / `Consolas`（按平台降级）

字体族**统一**通过设计 Token 暴露： `--font-display` / `--font-body` / `--font-mono`。

### 4.2 字体层级原则

- **字重分层** — 通过字重（400 / 500 / 600 / 700）建立层级，不依赖纯字号变化
- **标题收紧** — 标题字距 `letter-spacing: -0.01 ~ -0.02em`，提升密度感
- **正文松弛** — 正文行高 ≥ 1.5，确保长时间阅读舒适
- **非标签文字不全大写** — 大写仅用于注释、徽章、表头（`text-transform: uppercase` + `letter-spacing` 提升辨识度）
- **字体使用反模式**：禁止使用通用衬线体（Times、Georgia 等）；系统回退字体仅作降级

## 5. 组件设计原则

本章定义组件的**通用设计原则**，具体实现见 `design/components/`。

### 5.1 通用原则

- **状态可见性** — 每个交互组件**必须**具有：默认 / 悬停 / 激活 / 禁用 / 焦点五种状态
- **焦点环强制** — 焦点指示器**始终可见**，使用主强调色，**禁止移除**
- **触控目标** — 所有交互元素触控区域最小 44×44 CSS 像素
- **数据呈现** — 数字、ID、时间戳、路径、版本号、命令等结构化数据**使用等宽字体**
- **状态颜色** — 状态颜色**必须**配合图标或文字标签
- **玻璃层级清晰** — 每个组件**必须**明确归属于 §3.1 中的一级玻璃面，**禁止**混用层级

### 5.2 组件类型边界

- **按钮层级** — 至少区分：主要操作（`primary`，实心深色）、次要操作（`secondary`，玻璃 + 描边）、幽灵操作（`ghost`，透明）、危险操作（`error` 色调）四级
- **卡片层级** — 标准容器（`.card`）与数据展示容器（`.stat-card`）可复用，差异通过内容密度与是否启用 tint 氛围层实现
- **输入框层级** — 文本输入与搜索输入共用基础规范，差异通过功能性扩展（图标前缀、清除按钮）实现
- **导航层级** — 侧边栏（桌面）与底部 tab bar（移动 ≤768px）承担不同的导航职责，**不混用**；断点切换时**必须**保持当前视图状态
- **Chip 系列** — 至少区分：选择类（`target-chip`，单击切换）、状态类（`group-chip`、单选激活）、同步类（`cli-sync-chip`，三态：unsynced / syncing / synced）、过滤类（`tab-item` / `source-tab`，互斥激活）
- **状态徽章** — 状态指示器（圆点 + 文字）必须语义清晰，不依赖颜色单独传达；提供 `success` / `warn` / `error` / `info` / `progress` / `outline` 六类
- **操作阶段徽章** — 长时间操作（preparing / downloading / installing / verifying / completed / failed / cancelled）使用统一 `op-stage` 样式，与短状态徽章**视觉分离**

### 5.3 禁止模式

- 自定义鼠标指针
- 输入框浮动标签
- 通用"暂无数据"空状态（需根据上下文定制）
- 圆形旋转加载器（优先使用骨架屏；CLI 同步 chip 的 `syncing` 状态允许**仅在徽章内部**使用小尺寸旋转）
- 玻璃面与玻璃面直接相邻（**必须**留有 ≥ 8px 透明间隙或分隔线，避免模糊叠加）
- 将 tint 氛围色用于非数据展示容器

## 6. 布局原则

### 6.1 网格与节奏

- **主内容区**采用 12 栏网格制
- **间距体系**为多级递进（xs / sm / md / lg / xl / 2xl / 3xl），形成 4px 基线节奏
- **最大内容宽度**居中约束，超出宽度进入留白区
- **侧边栏宽度** `240px`、**顶部栏高度** `64px` 为固定 Token（`--sidebar-w` / `--topbar-h`）

### 6.2 响应式策略（v3 四档断点）

布局响应不同视口宽度采用**渐进式栅格切换**，而非等比缩放：

| 断点            | 宽度范围         | 关键变化                                                                 |
| --------------- | ---------------- | ------------------------------------------------------------------------ |
| **宽视口**      | ≥ 1024px         | 完整布局：侧栏 + 顶部栏 + 4 栏 stat / 3 栏 marketplace / 完整过滤器并排 |
| **平板**        | 768 ~ 1023px     | stat 4 栏 → 2 栏，settings 双列 → 单列，agents 网格收紧；侧栏仍保留      |
| **小屏 / 大屏手机** | 480 ~ 767px | **侧栏隐藏**，底部 tab bar 出现；过滤栏改为垂直堆叠                      |
| **窄视口**      | < 480px          | stat 单列；卡片 padding 收紧；按钮 padding / 字号缩小                    |

> 断点切换**不得**破坏当前视图状态：用户从桌面缩小到手机时，所选视图**保持**，仅布局重构。

### 6.3 布局边界

- 主内容区**禁止**使用绝对定位
- 禁止元素重叠
- 禁止居中 Hero 区块（统一左对齐以保证扫描效率）
- 禁止 3 等分卡片网格（采用 2 栏或非对称布局）
- 禁止使用 Flexbox `calc()` 技巧进行布局计算
- 移动端底部 tab bar **必须**限制为 5 个以内的核心入口（避免拥挤），其余导航通过完整菜单或侧滑抽屉进入

## 7. 动效与交互原则

### 7.1 时长分级

动效时长分四级：`fast` (150ms) / `base` (200ms) / `slow` (300ms)，基础缓动曲线 `cubic-bezier(0.4, 0, 0.2, 1)`。

### 7.2 交互反馈原则

- **悬停反馈** — 允许的反馈形式：颜色变化、玻璃透明度提升、`translateY(-2px ~ -3px)` 微抬升 + 阴影扩散；**禁止**使用缩放（scale）或旋转（rotate）作为悬停效果
- **加载反馈** — 优先骨架屏（`.skeleton`），禁用全屏连续旋转加载器；CLI 同步 chip 的 `syncing` 状态徽章允许**小尺寸**连续旋转
- **页面过渡** — 淡入为主，列表类内容可使用交错延迟（每项 ≤ 50ms）
- **焦点反馈** — 焦点环过渡时长与悬停同步，不闪烁
- **按钮按下** — `primary` 按钮按下时 `translateY(0)` 回正 + 背景色加深一档（`--accent-press`）

### 7.3 数据卡氛围动画（v3 允许的持续动画）

数据展示容器（`.stat-card.tint-*`）允许两类**极慢速**持续动画，**仅作用于氛围层**（`:before` / `:after` 伪元素），不影响内容：

- **Tilt Drift**（`tint-drift`）— 径向渐变在 8s 周期内做 ≤ 10% 位移 + ≤ 1° 旋转，制造柔和的光晕流动
- **Sweep**（`tint-sweep`）— 顶部高光带在 4.5s 周期内横扫一次，模拟玻璃面光线反射

约束：

- 仅作用于 `position: absolute` 伪元素层，**不改变布局**
- 动画必须**可被 `prefers-reduced-motion` 关闭**
- 4 种 tint 类型的动画**错峰启动**（`-2s` / `-4s` / `-6s` 延迟），避免视觉同步
- 内容层（数字、标签、副标题）**永远静止**

### 7.4 动效反模式

- 禁止在 UI 元素上使用弹跳缓动
- 禁止视差滚动
- 禁止元素随滚动淡出/缩放
- 禁止装饰性动画（无功能目的的动画）

## 8. 主题基线与变体

v3 不再强制"暗色 / 亮色"二分。每种主题**独立**携带其色温、明度、玻璃透明度参数。本章仅规定**两类基线**的通用约束。

### 8.1 浅色基线（默认基线）

- **核心特色** — 玻璃态（Glassmorphism）效果：半透明背景 + 背景模糊 + 微妙阴影
- **色温** — 由主题决定（warm / cool / neutral / expression 任一）
- **强调色** — 通常为低饱和度的暖色或近黑色，**不使用**冷色强调（蓝、紫）作为唯一强调（**例外**：`aurora` / `cyberpunk` 等表达性主题允许）
- **背景** — 允许使用多色径向渐变 + 细网格纹理 + 微弱噪点（`body::after` 噪点层 opacity ≤ 0.30）组合

### 8.2 暗色基线

- **层级构建** — 通过亮度提升建立层级，层级越高，背景越亮（玻璃面透明度更高）
- **玻璃面** — 使用**白色叠加**而非黑色叠加来构造玻璃（`rgba(255,255,255,0.04 ~ 0.10)`），避免暗色主题发灰
- **文字层级** — 通过**明度梯度**而非色相变化构建
- **强调色** — 保留与浅色基线同色相的强调色，但通常提高亮度与饱和度
- **噪点层** — 暗色基线下 opacity 降至 ≤ 0.15

### 8.3 跨基线一致性

- 同名主题（如 `warm` 浅色 vs `warm` 暗色版本）**保持**色相家族一致
- 数据卡 tint 体系在两类基线下**均可用**，仅调整不透明度与噪点强度
- 焦点环、错误、警告、成功的语义色**不随基线翻转**

## 9. 图标系统原则

### 9.1 图标分类

图标分三类：

- **应用图标** — 产品视觉标识，跨平台一致
- **界面图标** — 导航、操作、状态、文件类型
- **品牌图标** — CLI 工具等第三方品牌资产

### 9.2 设计原则

- **统一线条风格** — 同一界面内不得混用线条图标与填充图标（**例外**：应用图标与品牌图标可填充）
- **语义一致性** — 同一概念在不同上下文使用相同图标
- **品牌图标中立** — 第三方品牌图标忠实于原品牌识别，不做品牌重塑
- **主题响应** — 界面图标使用 `currentColor` 响应主题，品牌图标保留固定品牌色
- **尺寸分级** — 标准 / 小 / 大 三档，线宽随尺寸分级调整
- **状态指示** — 圆形 spinner / 三角形警告等纯几何符号可直接作内联状态符号（v3 允许）

### 9.3 禁止模式

- 图标上添加文字或编号
- 使用非项目品牌色
- 混合线条与填充两种风格
- 直接修改自动生成的平台图标文件

## 10. 无障碍设计原则

无障碍是**强约束**，贯穿所有设计决策，**不**作为优化项。

### 10.1 视觉无障碍

- **色盲友好** — 状态信息不得仅依靠颜色传达
- **对比度合规** — 满足 WCAG AA 等级，关键 UI 达 AAA
- **焦点可见** — 焦点指示器**始终可见**
- **持续动画可关闭** — 所有持续动画（`tint-drift` / `tint-sweep` / 旋转徽章）必须响应 `prefers-reduced-motion`

### 10.2 操作无障碍

- **键盘完整可达** — 所有交互通过键盘可完成
- **触控目标** — 最小 44×44 像素
- **跳转链接** — 提供跳转到主内容的快捷链接
- **模态焦点管理** — 打开模态时焦点进入模态，关闭时返回触发元素；`Escape` 键关闭模态与确认对话框

### 10.3 语义无障碍

- 语义化 HTML 结构优先
- 仅图标按钮**必须**添加 ARIA 标签
- 自定义组件**必须**添加 `role` 属性

---

## 11. 内容与文案原则

### 11.1 文案风格

- **专业克制** — 避免营销词汇（"提升"、"无缝"、"释放"、"次世代"等）
- **准确优先** — 数据描述精确，不使用虚假精度（`99.99%`、`50%` 等圆整数）
- **场景化** — 空状态、错误状态等根据上下文定制，**禁止**使用通用"暂无数据"
- **确认对话** — 危险操作必须显示**具体影响**（"Delete 12 plugins? This will also unlink 3 MCP servers."）而非通用"Are you sure?"

### 11.2 内容反模式

- 禁止使用 emoji
- 禁止使用占位符（"张三"、"某公司"）
- 禁止 Lorem ipsum 占位文字
- 禁止使用通用图库照片
- 禁止失效的链接或图片

---

## 12. 通用反模式（全部主题与基线适用）

| 类别     | 反模式                                                       |
| -------- | ------------------------------------------------------------ |
| **字体** | 装饰性渐变标题、非标签文字大写、依赖纯字号分级               |
| **色彩** | 纯黑/纯白表面、单一界面多强调色、Tint 用作强调               |
| **玻璃** | 玻璃面与玻璃面紧贴、纯白玻璃面、玻璃面 + 锐利硬边框         |
| **布局** | 元素重叠、绝对定位、3 等分卡片、居中 Hero、Flexbox 布局 hack |
| **组件** | 自定义光标、浮动标签、通用空状态、全屏旋转加载器             |
| **动效** | 弹跳缓动、视差、装饰性动画、内容层持续动画                   |
| **内容** | emoji、营销词汇、虚假精度、占位符、图库照片                  |

## 13. 适用范围与维护说明

### 13.1 适用范围

本规范适用于 Forge 产品的：

- 桌面应用（**Tauri 跨平台构建**，macOS / Windows / Linux）
- 移动端响应式视图（同一份代码库，可降级布局）
- 配套的设计系统（组件库、图标库、主题包）
- 官方网站、文档站点

### 13.2 不适用范围

- 内部工具与脚本（仅参考风格）
- 第三方集成的品牌内容
- 临时性的营销/活动页（需另行评审）

### 13.3 配套文档

- **设计 Token**：`design/tokens/` — 具体色值、字号、间距、阴影、动画曲线、玻璃透明度
- **主题清单**：`design/tokens/themes/` — 预置主题定义（≥ 20 套）
- **组件规范**：`design/components/` — 组件状态、变体、API
- **图标资产**：`design/icons/` — 图标源文件、生成脚本
- **原型与高保真**：`design/wireframes/`、`design/high-fidelity/`
- **参考原型（v3 基线）**：`design/forge-cross-platform-glass.html`

### 13.4 变更原则

- 规范变更需设计总监审批
- Token 变更需同步更新代码与文档
- 新增主题需在 `design/tokens/themes/` 注册并通过 §10 的无障碍审计
- 不向下兼容的破坏性变更需在 changelog 中明确标注
